# Platform QA Evidence Collector Requirements

## 목적

QA 중 이슈 발생 시 사용자가 별도 정리 작업 없이 스크린샷, 영상, 로그, 디바이스 정보를 수집하고, 지정된 Jira 프로젝트에 티켓으로 자동 등록하는 멀티 플랫폼 증적 수집 도구를 개발한다.

대상 플랫폼은 PC, Android, iOS이다. Android는 PC에서 ADB 또는 미러링으로 제어하는 방식이 아니라 APK 내부에서 독립적으로 동작해야 한다.

## 현재 PC 수집기 기준 기능

현재 `qa-issue-collector`는 아래 기능을 기준 모델로 한다.

- Jira URL, 계정, API Token 설정
- Jira 프로젝트 선택
- 이슈 타입 선택
- 담당자 선택
- 우선순위 선택
- Test Environment 선택
- Reproducibility 선택
- Component 선택
- Label 입력
- 이슈 요약 입력
- 재현 절차, 실제 결과, 기대 결과 입력
- 디바이스 정보 수집
- 스크린샷 수집
- 영상 수집
- 버튼 클릭 기준 이전/이후 로그 수집
- Jira 티켓 생성
- 수집 파일 Jira 첨부

## 우선순위

1순위:

- 녹화/증적 관리
- 로그 수집
- Jira 티켓 자동 생성
- 스크린샷, 영상, 로그, 디바이스 정보 첨부

2순위:

- PC 연동
- LLM 기반 이슈 내용 정리
- 자동 재현 감지
- 통계/대시보드

## 공통 요구사항

모든 플랫폼은 아래 기능을 공통으로 제공해야 한다.

- Jira 설정
  - Jira URL
  - 계정
  - API Token 또는 OAuth
  - 프로젝트 선택
  - 이슈 타입 선택
  - 필수 필드 조회
  - 프로젝트별 필드 매핑 저장
- Jira 이슈 필드
  - Summary
  - Description
  - Project
  - Issue Type
  - Assignee
  - Priority
  - Component
  - Labels
  - Test Environment
  - Reproducibility
  - Device Environment
  - Attachments
- 증적 수집
  - 스크린샷
  - 영상
  - 로그
  - 디바이스 정보
  - 앱 정보
  - 수집 시각

## 버튼 타입

두 가지 주요 액션 버튼이 필요하다.

### 스크린샷 이슈 등록

1. 사용자가 스크린샷 버튼 클릭
2. Jira 티켓명 입력
3. 현재 화면 스크린샷 수집
4. 디바이스 정보 수집
5. 앱 정보 수집
6. Jira 티켓 생성
7. 스크린샷 및 데이터 첨부

### 영상 이슈 등록

1. 사용자가 영상 버튼 클릭
2. Jira 티켓명 입력
3. 영상 녹화 시작
4. 녹화 중 로그 수집
5. 녹화 종료
6. 디바이스 정보 수집
7. 앱 정보 수집
8. Jira 티켓 생성
9. 영상, 로그, 데이터 첨부


```text
4. Jira 티켓명 입력
5. 영상 녹화 시작
```

## 첨부 파일 규칙

티켓명을 기준으로 파일명을 생성한다.

예시 티켓명:

```text
로그인 버튼 클릭 시 앱 종료
```

첨부 파일:

```text
로그인_버튼_클릭_시_앱_종료.png
로그인_버튼_클릭_시_앱_종료.mp4
로그인_버튼_클릭_시_앱_종료.log
device_info.json
```

## Android 요구사항

Android는 반드시 APK 내부에서 동작해야 한다.

필수 기능:

- APK 내 스크린샷 캡처
- APK 내 화면 녹화
- 앱/디바이스 정보 수집
- 로그 수집
- Jira API 호출
- Jira 티켓 생성
- 첨부 업로드

주의사항:

- Android 일반 APK는 다른 앱의 `logcat` 전체 접근이 제한될 수 있다.
- 테스트 대상 앱의 로그를 안정적으로 수집하려면 아래 중 하나가 필요하다.
  - 테스트 대상 앱 내부 로그 SDK 연동
  - 앱 로그 파일 저장 후 수집
  - 디버그 빌드 전용 로그 접근
  - MDM/사내 테스트 권한
  - 접근성/MediaProjection 기반 보조 기능

## iOS 요구사항

iOS도 앱 내부에서 동작하는 형태를 목표로 한다.

필수 기능:

- 화면 캡처 또는 사용자가 선택한 이미지 첨부
- 화면 녹화 또는 시스템 녹화 파일 첨부
- 디바이스 정보 수집
- 앱 정보 수집
- Jira API 호출
- Jira 티켓 생성
- 첨부 업로드

주의사항:

- iOS는 Android보다 로그/화면 녹화 제약이 강하다.
- 타 앱 로그 수집은 일반 앱 권한으로 거의 불가능하다.
- 테스트 대상 앱에 로그 SDK 또는 내부 리포트 기능 연동이 필요할 수 있다.

## PC 요구사항

PC 버전은 현재 `qa-issue-collector`를 기준으로 고도화한다.

- 현재 기능 유지
- Jira 필드 매핑 개선
- 스크린샷/영상/로그 파일명 규칙 통일
- Jira 프로젝트별 설정 저장
- EXE 패키징
- 향후 LLM 이슈 정리 옵션 추가 가능

## 개발팀 전달 핵심 문장

현재 PC 기반 QA Issue Collector에서 구현한 Jira 프로젝트/필드 설정, 증적 수집, Jira 티켓 생성 구조를 기준으로 PC, Android, iOS 플랫폼별 독립 클라이언트를 개발한다.

특히 Android는 PC 미러링/ADB 기반이 아닌 APK 내부 동작 방식이어야 하며, 최우선 기능은 스크린샷/영상/로그/디바이스 정보를 수집해 Jira 티켓으로 자동 전송하는 것이다.

스크린샷 등록과 영상 등록은 별도 버튼 플로우로 제공되어야 하며, 티켓명 입력 후 각 플랫폼에서 수집 가능한 증적과 디바이스 정보를 Jira의 지정 필드 및 첨부파일로 등록해야 한다.
